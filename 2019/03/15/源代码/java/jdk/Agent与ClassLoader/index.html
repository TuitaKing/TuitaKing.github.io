第二回 KClass – OOP 模型

为什么最开始的时候java是被称呼为解释型语言的呢，就是因为他其实是把你写的Java 代码转换成某些特殊的c++代码（个人愚见）。
其中OOP（ordinary object pointer），普通对象指针？因为java中没有指针的概念，但是我们的对象最终是生成在内存中的，这个可以理解为一个对象生成后的内存位置，也就是这个对象（说点废话）。这个普通对象指针指向的就是描述对象实例信息。可以理解为每生成一个对象就会创建一个oop 具体可见，原文举了例子，一个java的实例对象（instanceOopDesc），里面的域都是由程序员定义的，为了让class文件和这个对象之间保持对应关系，会在里面加一个header，来标注class对象（instanceKlass）。顶级class为oopDesc（）

可以看到oopDesc作为基类，包含了对象头，对象头最常见就是用来锁synchronized的实现。这也就可以说明为什么.class的锁是全局锁，而对象的锁只是使用当前对象的锁了。（BYW：前文忘记说了，同一个class之所以只能被classloader加载一次，是因为classloader会被注册到class上，后续将agent可能会深入，这里只是想说明下为啥.class是唯一的，当然，引出一个问题，两个不同的classloader来加载相同的class的话，这个时候.class 的锁是加在哪个上面的？）
上文提到，static方法适合Class绑定的，在这里也可以看出为什么我们在使用静态方法的适合不需要实例，因为两者在jvm里的抽象就是完全不同的类。（个人理解）
其实整个KClass–OOP模型还有很多的细节，包括方法的抽象等等，都可以深入去了解下。这里按下不表。

第三回 java的agent

最开始的agent是用c或者c++写的，其实看到上面，就应该会想，既然最后都是映射成c++了，那么我用c写个方法，是不是也可以用java来调用，理论上是可以的如同native或者android编译c为.o后也能够调用（大学第一个通宵就是搞这个啊……^_^）。那么，我们又是如何通过写-javaagent就能够让他生效的呢？
在jvm启动的时候，会将解析参数，如果有agent相关的参数（-javaagengt etc），会将做一个钩子函数或者说注册事件监听，等到classLoad的时候进行处理。
如果在vm启动过程中加载agent，那么会在vm初始化过程中先执行libinstrument.dylib里InvocationAdapter.c的Agent_OnLoad方法，这里主要是实例化agent，解析agent的MF文件，将相关属性取出来，并注册jvmti的一些回调函数，在vm初始化完成之后，会通过回调函数去实例化Instrumentation实现对象，设置ClassFileLoadHook函数，并调用Pre-Main指定类的premain方法。

下面是对于的函数。

总之，定义好了Premain-Class方法我们一般会将定义的agent添加到Instrumentation中去，等到上诉的回掉函数被触发后，会进入到sun/instrument/InstrumentationImpl中执行transform方法，这个就是我们写入java的地方。
还有一种是启动后可以将当前jvm运行的方法attach到目标jvm中，这个其实是jvm启动了一个listerner线程，是在jvm启动的时候开启的，具体的代码我就不再贴了。但是，我们其实常常看到这个东西，比如dump，就是本地的jvm attach到目标jvm中了。包括阿里开源的arthas，我们使用的bytebuddy都是使用的后者attach-on。可以看下byte-buddy的Attacher，或者arthas的agent部分实现。这种方法也会触发上文提到的transformClassFile，可以修改一些class字节码，不过不能修改原有方法的参数，方法标签，继承关系等等。所以，目前包括Tomcat的热更新就是使用webclassLoader来进行动态监听和加载，而不能做到不卸载当前class直接修改，R大的解释是说，动态修改的成本太大，毕竟线上肯定有跑着的jvm，他们的实例对象如果都要修改的话，会有反优化的问题，不做深入的探讨。

关于Byte-Buddy

我们目前的agent是基于Byte-Buddy来做的，Byte-Buddy算是在动态字节码修改业界最好的了，他封装了ASM，ASM是动态修改字节码的基石，通过对class，method的封装，使用vistor设计模式，有点深入理解字节码，篇幅有限，感兴趣可以自己去看下官网，他的设计文档写的很清楚。

附 arthas

arthas 是阿里开源的Java诊断工具。他的原理就是Agent_OnAttach，然后通过字节码增强，将线程栈帧进行插入实现debug，通过retransformation方法来dump类（jad）。

顺便提一句，agent的jar会被bootstrap所加载，也就是说jar中的类会被bootstrapclassloader所加载。
